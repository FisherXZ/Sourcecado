What Is Built

1. Ingest Read exported markdown, text, CSV, and email files into source records and chunks.
2. Chunk Create citation-backed memory chunks with hashes and deterministic embeddings.
3. Extract Use deterministic CSV extraction or LLM-backed structured extraction for unstructured files.
4. Refresh Cache extraction runs, merge aliases, dedupe facts, flag conflicts, and mark stale memory.
5. Ask Answer sourcing-history questions with Answer, Evidence, Gaps, and Next Action.

Key Contracts

Answer contract

Every sourcing answer must include these four sections:

  • Answer: direct sourcing-memory synthesis.
  • Evidence: source citations for factual claims.
  • Gaps: missing, candidate, conflicted, or stale memory.
  • Next Action: the next useful sourcing step.

Extraction cache contract

Extraction runs are cached by chunk hash plus extractor type, extractor version, prompt hash, schema version, and model name. This lets unchanged chunks reuse previous extraction outputs while invalidating stale extractor behavior.

Clean memory rules

  • High-confidence non-conflicting facts become accepted.
  • Low-confidence facts stay candidate.
  • Same subject and predicate with different objects become conflicted.
  • Previously accepted facts missing in later refreshes become stale gaps.

Local-only boundary

The MVP is single-user local memory. It does not enforce app-level permissions yet, so source material should only be ingested when the user is already allowed to use it.

How It Runs

The documented local workflow is:

npm install
npm run build
npm run sourcyavo -- ingest seed-data/
npm run sourcyavo -- refresh
npm run sourcyavo -- ask "Who needs follow-up for AI safety?"

The local SQLite database lives at .sourcyavo/memory.db. CSV-only refresh works locally without model configuration. Markdown, text, and email refresh use the LLM extractor and require OPENAI_API_KEY plus SOURCYAVO_LLM_MODEL.

Runtime note

The default Codex bundled Node 24 runtime could not load the existing better-sqlite3 native binary because it was built for Node ABI 131. Verification passed under the Homebrew Node 23.7.0 runtime that matches the installed native module.
