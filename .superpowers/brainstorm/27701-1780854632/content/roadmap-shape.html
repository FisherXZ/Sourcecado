Proposed Roadmap Shape

Small checkpoints. Each one proves a user workflow and one memory-system idea.

SourcyAvo Roadmap: Grounded Checkpoints
Checkpoint 0

Current CLI Baseline

Local ingest, refresh, and ask over exported files. Four-section answers with citations and gaps.

Checkpoint 1

Real Data Stress Pass

Run broad real Codeology source material through the local pipeline. Use failures to define the actual MCP/read requirements.

Checkpoint 2

Read-Only MCP Memory Tools

Expose ask/search/source/gap tools to Codex or Claude. Admin actions stay CLI-only.

Checkpoint 3

Source-Scoped + Temporal Answers

Answer “what did we know as of date X?” and enforce source scope before retrieval.

Checkpoint 4

Thin Research Chat

A simple UI for Sourcing Directors, built on the same read surface as MCP.

Design Principle

No grand platform. Each checkpoint should be demoable with real sourcing questions and should retire one specific risk.
