Grounded SourcyAvo Roadmap Options

All three stay simple. The question is which risk we retire first.

A

MCP-First Thin Surface

Expose read-only MCP tools quickly, then deepen temporal/source-scoped behavior as the agent surface reveals gaps.

Good

  • Fastest agent-native checkpoint
  • Clear path to Claude/Codex usage
  • Keeps UI deferred until memory is useful

Risk

  • Temporal correctness may lag behind the surface
  • Can become a wrapper around today's CLI if underspecified
B

Temporal Core First

Upgrade facts to Graphiti-inspired validity windows before MCP becomes a serious surface.

Good

  • Best foundation for “as of date X” questions
  • Makes stale/conflicted/superseded behavior rigorous
  • Deepest memory-system learning value

Risk

  • Less visible progress early
  • Could overbuild before real agent workflows shape it
C

Reference-Backed Checkpoints

Ship in small checkpoints where each release has a user surface and one serious memory primitive.

Good

  • MCP v1 still lands early
  • Temporal/source scoping become testable requirements
  • Research Chat arrives after agent usage proves the shape

Risk

  • Requires discipline to keep each checkpoint small
  • Broad real data means more messy edge cases from day one

Recommendation

Pick C: read-only MCP first, but define MCP success around source-scoped and temporal answers, not merely “the CLI can be called by an agent.”
